# Admin Rebuild Addendum

This addendum overrides earlier guidance that suggested keeping the existing admin dashboard if usable.

## Final decision
The project will use a **fresh admin dashboard**.

## Implication
The rebuild should assume:
- new frontend
- new admin dashboard
- new content model implementation
- content migration only after the new schema is stable

## Rule for implementation agents
Do not inherit old admin assumptions blindly.
Use the current live site and previous dashboard only as reference material for:
- content inventory
- migration source
- copy review
- asset collection

They are **not** the foundation of the new system.

## Priority
The new admin should be designed to support the website's new conceptual structure:
- Sight
- Sound
- Touch
- Voice
- Dream
- Heart
- Pulse

and the content types defined in the content model and entry guide.
