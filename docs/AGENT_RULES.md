# Agent Rules and Guardrails

This file defines non-negotiable working rules for Antigravity and any design/coding agents touching the WEI IN SIGHT rebuild.

## 1. Preserve the concept
Do not flatten the product into a normal portfolio layout.

The homepage must remain:
- split-screen on desktop
- text/navigation on the left
- body-based navigation on the right
- poetic-first in tone
- cinematic in mood

## 2. Preserve prior stage work
Every stage must build on the existing codebase.

Do not:
- restart from scratch
- replace the entire routing structure
- discard shared components
- rename established sections without reason
- break the body-anchor metaphor
- undo previous stage decisions unless explicitly required

## 3. Keep naming consistent
Use these canonical main labels only:
- Sight
- Sound
- Touch
- Voice
- Dream
- Heart
- Pulse

Use these mapping rules only:
- Eyes = Sight
- Ears = Sound
- Hand/Fingertips = Touch
- Throat = Voice
- Forehead/Temple = Dream
- Chest/Heart = Heart
- Wrist = Pulse

Do not invent extra primary categories unless explicitly requested.

## 4. Keep motion restrained
Motion should be:
- slow
- intentional
- smooth
- atmospheric

Motion should never be:
- flashy
- hyperactive
- arcade-like
- gimmicky

## 5. Keep the body symbolic
The body is a poetic navigation device.

It must not feel:
- clinical
- anatomical in a medical way
- fetishized
- robotic in a cold sci-fi way

Use stylized abstraction, elegant wireframe rendering, and minimal signal effects.

## 6. Protect usability
The body is not the only navigation method.

Always maintain:
- fully usable left-side text navigation
- route-based dedicated pages
- mobile-friendly menu behavior
- keyboard-accessible core navigation
- reduced-motion fallback
- useful loading and fallback states

## 7. Use Option B only
Section destination behavior must use dedicated pages.

Do not build the site so that all content lives only inside homepage overlays.

Homepage = gateway  
Destination pages = deep content

## 8. Respect the visual system
Primary palette:
- near-black / midnight
- neon pink
- white
- subtle magenta/violet atmospheric depth only if restrained

Do not shift to:
- rainbow neon
- bright cyan-heavy cyberpunk
- oversaturated gradients
- light-mode-first homepage

## 9. Respect the tone
The site voice should feel:
- poetic
- mature
- elegant
- personal
- contemporary
- precise

Do not write copy that feels:
- generic
- corporate
- overly techy
- try-hard mystical
- empty luxury jargon

## 10. Google Stitch MCP usage
Use Google Stitch MCP for:
- spacing refinement
- layout hierarchy
- typography balance
- visual polish
- responsive design refinement
- interface consistency

Do not use Google Stitch MCP to erase or replace:
- the core navigation metaphor
- the route structure
- body-anchor logic
- core interaction blueprint

## 11. Performance rules
3D must be optimized.

Required:
- lazy loading
- fallback states
- mobile simplification
- reduced-motion support
- asset optimization
- no heavy unbounded particle systems
- no unnecessarily dense line mesh

## 12. Content rules
Content pages must be data-driven where possible.

Reusable templates should exist for:
- works
- writing
- music
- exhibitions/features
- collector/commission/contact items

## 13. Decision priority
When tradeoffs happen, prioritize in this order:
1. clarity of concept
2. usability
3. performance
4. visual beauty
5. extra visual effects

## 14. Final quality test
Before considering any stage complete, check:
- Does it still feel like WEI IN SIGHT?
- Does it still feel human?
- Is the interaction elegant?
- Is the product still clear?
- Did this stage preserve what previous stages built?
